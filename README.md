# Performance-Portal

## Description

The Performance Portal is a data-driven dashboard, built in Power BI, designed to transform operational performance meetings into insight-led decision-making forums.

Originally developed from monthly performance board meetings, the project evolved into a centralised platform that surfaces actionable insights, improves visibility of provider performance, and enables more targeted operational interventions.

The solution leverages SQL-based data modelling and automated reporting to ensure accuracy, scalability, and accessibility across the business.  


## Current Business Problem

Performance board meetings were primarily operational discussions, with limited use of data to proactively identify issues or opportunities.  

**Challenges included:**  

	• Meetings lacked a structured, insight-led narrative  
	• Existing dashboards did not surface actionable opportunities  
	• Reactive rather than proactive operational decision-making  	
	• Limited visibility of provider-level performance across the business  
	• Diagnostic data was underutilised and inconsistently applied  

## Solution

To address these issues, the project introduced both a cultural and technical transformation:  

**Reframing Performance Boards**  
	•	Shifted meetings from operational updates to data-led performance discussions  
	•	Introduced a structured approach focused on identifying opportunities and driving action  
	•	Enabled leadership to make decisions based on evidence rather than anecdote  

**Targeted Opportunity Identification**  
•	Developed a focused methodology using SQL to identify:  

	  - Low diagnostic scores  
	  - No recorded barriers  
	  - Recently employed  
	  - Recently engaged    
• This created a clear and actionable pipeline for operational teams to prioritise activity  

**Performance Portal Dashboard (Power BI)**  
	•	Built a centralised Power BI dashboard to:  
  
	  - Surface previously hidden insights  
	  - Track key performance indicators across providers  
	  - Enable drill-down into operational and participant-level data  
• Data models were optimised using SQL transformations and a Power BI star schema to ensure performance and scalability  
• Designed to support both strategic oversight and day-to-day decision-making  

**Automated MI Workbook Integration**  
	•	Integrated an automated MI (Management Information) workbook into the portal  
	•	Backed by SQL pipelines to ensure consistent and reliable data refreshes  
	•	Provided company-wide visibility of provider-level performance  
	•	Reduced manual reporting effort and improved data accessibility  

**Diagnostic Framework Improvement**  
	•	Identified a pattern of participants with low diagnostic scores not progressing into work as expected  
	•	Investigated underlying data using SQL analysis  
	•	Discovered the diagnostic questionnaire was not being used effectively (e.g. box-ticking rather than meaningful input)  
	•	Implemented a review and redesign of the questionnaire and improved guidance on how it should be applied operationally  

## Impact

Delivered measurable improvements in data visibility, decision-making, and operational efficiency:  

• Transformed performance boards into insight-driven decision-making forums  
• Enabled targeted operational interventions based on clear data signals  
• Improved visibility of provider performance across the organisation  
• Reduced reliance on manual reporting through automation  
• Strengthened data quality through improvements to the diagnostic process  
• Increased effectiveness of engagement strategies for participants with low diagnostic scores
